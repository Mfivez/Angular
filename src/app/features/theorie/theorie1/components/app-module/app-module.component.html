6. Le module app :

Le module app est le module racine de l'application, celui qui englobe l'ensemble du projet Angular. Il est comparable à la fonction main en Java et sert de point d'entrée pour le projet.

app.module.ts :

  • Le décorateur @NgModule : Annoté pour définir la configuration du module via différents attributs (comme declarations, imports, providers, etc.).

  • declarations : Liste les composants utilisés par le module.

  • imports : Spécifie les modules externes ou autres modules Angular dont le module courant dépend.

  • exports : Permet de rendre les composants, directives ou pipes d'un module disponibles pour d'autres modules.

  • providers : Déclare les services et autres dépendances disponibles dans ce module via l'injection de dépendances.

  • bootstrap : Spécifie le composant principal de l'application, généralement AppComponent.

  • export class AppModule : Le module est exporté et donc importable ailleurs dans l’application.

La partie component du module :

Composée de 3 fichiers distincts :

  • app.component.html : Le template visuel

  • app.component.scss : La feuille de style

  • app.component.ts : La logique des composants en TypeScript, incluant le décorateur @Component avec les attributs selector, templateUrl, et styleUrls.

Le fichier app-routing.module.ts :

Gère la navigation entre les différentes pages ou composants de l'application Angular. Il configure les routes pour diriger l'utilisateur vers les différents composants en fonction de l'URL.

  • Le décorateur @NgModule : Définit la configuration du module de routage.

  • RouterModule et Routes :

    • RouterModule : Configure la navigation dans l'application.

    • Routes : Un tableau d'objets définissant les différentes routes, chaque objet contenant une URL path et le composant associé.

  • imports : Inclut RouterModule.forRoot(routes) pour configurer les routes.

  • exports : Permet de rendre RouterModule accessible à l'ensemble de l'application.

Qu'est-ce qu'un module de routage ?

Le module de routage en Angular permet de gérer la navigation entre les différentes vues ou composants de l'application. Il simplifie la gestion des chemins, des composants correspondants et des paramètres d'URL, facilitant ainsi la création d'applications à page unique (SPA).
