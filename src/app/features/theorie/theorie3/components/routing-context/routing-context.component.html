Contexte :

Lorsqu'on crée des modules, il est nécessaire de configurer le routing de ceux-ci afin de pouvoir naviguer entre les pages qu'ils offrent

Pour se faire, nous allon devoir mapper une URL à un ou plusieurs composants de notre application.

Là, où, jusqu'à maintenant, vous n'aviez accès qu'à l'adresse http://localhost:4200, vous pourriez grâce au routing, décider de lier une adresse telle que http://localhost:4200/home à un composant home ou encore http://localhost:4200/demos à une page démo, elle même possèdant plusieurs démonstrations sur les pages http://localhost:4200/demos/i
